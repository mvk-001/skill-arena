# Artifact Structure

Layout and purpose of files produced by a Skill Arena compare run.

## Directory layout

```text
results/<benchmark-id>/<timestamp>-compare/
├── promptfooconfig.yaml
├── promptfoo-results.json
├── summary.json
├── workspace/
│   ├── <profile-id>--<variant-id>/
│   │   ├── (materialized workspace files)
│   │   └── .skill-arena/
│   │       └── hooks/
│   │           └── execution-events/
│   │               └── *.json
│   └── ...
└── merged/
    ├── report.md
    └── merged-summary.json
```

## File purposes

### promptfooconfig.yaml

The Promptfoo configuration generated by Skill Arena from the compare config.
Useful for debugging what the harness actually sent to Promptfoo.

### promptfoo-results.json

Raw Promptfoo output containing every evaluation cell, assertion result, and
provider response. Large file; use `summary.json` or `merged/report.md` for
most inspection tasks.

### summary.json

Normalized Skill Arena summary. Key sections:

- `providers`: maps provider ids to profile and variant metadata
- `scenarios`: per-prompt results with assertion outcomes
- `comparison.matrix`: structured pass/fail data per cell
- `comparison.profileSummary`: aggregate pass rates per profile

### workspace/

Materialized run directories, one per profile-variant combination.
Each contains the files the agent saw during execution.

The `.skill-arena/hooks/execution-events/` subdirectory captures observable
command invocations and tool-call streams when available.

### merged/report.md

Human-readable compare report. Contains:

- Profile comparison table with pass rates
- Per-prompt breakdown by profile and variant
- Unsupported or skipped cell annotations

This is the primary file to read when summarizing results.

### merged/merged-summary.json

Machine-readable merged summary. Same data as `report.md` in structured form.
Use when programmatic access to comparison data is needed.
