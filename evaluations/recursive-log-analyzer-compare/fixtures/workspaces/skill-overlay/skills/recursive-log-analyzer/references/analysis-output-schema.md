# Analysis Output Schema

This document defines the JSON structure generated by:
`skills/recursive-log-analyzer/scripts/run_recursive_log_analyzer.sh build`

## Top-level object

- `metadata` (object)
- `summary` (object)
- `root` (object, recursive node)
- `hotspots` (array of hotspot nodes)

## `metadata`

- `source_file` (string): Input file path.
- `source_lines` (integer): Number of input lines.
- `max_lines` (integer): Maximum segment size used for splitting.
- `min_lines` (integer): Minimum segment target used during splitting.
- `top_k` (integer): Number of hotspots stored.

## `summary`

- `total_lines` (integer): Total number of lines in input.
- `leaf_nodes` (integer): Count of terminal segments.
- `total_anomaly_score` (integer): Sum of leaf scores.
- `pattern_totals` (object<string, integer>): Aggregated keyword counts.
- `dominant_signals` (array<[string, integer]>): Top three pattern totals.

## Node object (`root`, descendants, and hotspots)

- `id` (string): Deterministic node id (`root`, `root.0`, `root.0.1`, ...).
- `depth` (integer): Recursion depth, where root is 0.
- `line_start` (integer): 1-based inclusive line number.
- `line_end` (integer): 1-based inclusive line number.
- `line_count` (integer): Number of lines in node range.
- `anomaly_score` (integer): Weighted score from `pattern_counts`.
- `pattern_counts` (object<string, integer>): Counts per anomaly signal.
- `preview` (string): Whitespace-normalized segment preview (truncated).
- `children` (array<node>): Child nodes, empty at leaves.

## Hotspot ordering semantics

`hotspots` are selected from leaves only and ranked by:
1. `anomaly_score` descending.
2. `line_count` descending for tie-breaking.
